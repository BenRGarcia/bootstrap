Git

Basic Commands memorize these...

Official reference for all things Git can be found here.

Day-to-Day Big Picture working alone...

  1. Create Repo on GitHub, clone to your machine
  2. Code a bunch
  3. Add files to staging
  4. Commit changes
  5. Push changes back to remote repo
  6. Perpetually epeat steps 2-5

In other words, after Repo created on GitHub

  1. $ git clone username@host:/path/to/repository

Then code a bunch in your text editor... then

  1. $ git add file-name-here
  2. $ git commit -m "commit-message-here"
  3. $ git push origin master

Day-to-Day Big Picture working with others...

  1. Pull down code from remote repo to local machine
  2. Create your own branch
  3. Code a bunch
  4. Commit changes
  5. Push changes back to remote repo
  6. Changes merged into remote master branch

Git Command Library of commonly used commands...

  • Initialize a Git Repo:
    $ git init
  • Clone a Git Repo:
    $ git clone url-of-repo-to-clone-goes-here
  • See the status of a Git Repo:
    $ git status
  • Add file(s) to staging:
    $ git add all or,
    $ git add file-name-goes-here
  • Remove file from staging (but keep code changes):
    $ git reset HEAD -- file-name-goes-here
  • Commit changes to master branch:
    $ git commit -m "Your-commit-remarks-go-here"
  • Push code to remote master branch:
    $ git push origin master
  • Pull in remote's code, master branch:
    $ git pull origin master
  • See what's different between 2 versions of code:
    $ git diff
  • Create a branch called "feature-name-here":
    $ git branch feature-name-here then,
    $ git checkout feature-name-here
    or the combo shortcut,
    $ git checkout -b feature-name-here
  • Undo all changes since the last commit you've made:
    $ git checkout -- file-name-goes-here
  • Merge changes (from master to branch, first checkout branch):
    $ git merge master
  • Tell Git which files to ignore (there should be many ignored files), add them to:
    $ vim .gitignore
  • One of the innumerable ways to see git commit history:
    $ git log
    or other fun ways:
    $ git log --graph --oneline --decorate --all
    $ git log --pretty=format:"%h - %an, %ar : %s"
    $ git log --pretty=format:"%h %s" --graph

Need-to-know Definitions

  • Repository - Container for snapshots and history, *not* a server
  • Remote - Connection to another repository somewhere in the world, like a URL
  • Commit - Basic unit of history. Contains author/date/time/comments/tree
  • Reference - A pointer to a Commit. Cheap, easy, to create, move, destroy
  • Branch - A kind of reference, when you add a commit to one, git moves it
  • Head - Points to the branch that youâ€™re on
  • Working directory - Sandbox, like any other set of files except that Git watches it
  • Index - aka staging area, proposal for next snapshot, changes are copied here from the working directory
  • Merge - a commit that combines two separate lines of history

Git Clone what happens?...

  1. A new folder named after the remote repo is created
  2. It is initialized as a Git repo
  3. A remote name origin is created, pointing to the URL your cloned from
  4. *All* of the repo's files and commits are downloaded there
  5. The default branch (usually called 'master') is checked out

Don't Index That!

Tons of files shouldn't be indexed by Git for a variety of reasons.

  • Add individual files to be ignored within the scope a specific git repo:
    $ vim .gitignore
  • Tell Git to globally ignore files (system wide):
    1. Create a .gitignore file in your home directory
    2. # Some good ideas for a global .gitignore file

# Mac
.DS_Store

# Windows
Thumbs.db

# WebStorm
.idea/

# vi
*~

# General
log/
*.log

# etc...
    3. Then, tell Git where the global .gitignore file is:
      $ git config --global core.excludesfile ~/.gitignore
